/* * STACK: variáveis locais — vida limitada à função * char buf[4096]; -> morre quando read_dict_file() retorna * char line[512]; -> morre quando parse_line() retorna * char val[256]; -> morre quando dict_lookup() retorna * * HEAP: malloc() — vida controlada pelo programador * char *filename; -> malloc(100) em get_dict_path() * char *dict; -> malloc(len+1) em read_dict_file() * char *nbr; -> malloc(40) em get_number() * * REGRA: todo malloc() tem um free() correspondente em run() * free(filename); free(dict); free(nbr); * * CONSEQUÊNCIA: usar char filename[100] em get_dict_path() * e retornar filename* causaria USE AFTER STACK — bug clássico C. */

Makefile Explicado

O Makefile automatiza a compilação. Clique em cada regra para ver a explicação detalhada.

Makefile — arquivo completo anotado
# ── Variáveis ─────────────────────────────────────── NAME = rush-02 # nome do executável gerado SRC = main.c group.c lookup.c parse.c \ utils.c rush_number.c rushfile.c rush_ini.c FLAGS = -Wall -Wextra -Werror # compilação estrita RM = /bin/rm -f # rm sem erro se não existir # ── Regras ────────────────────────────────────────── all: $(NAME) # regra padrão $(NAME): gcc $(FLAGS) -o $(NAME) $(SRC) # compila tudo direto clean: $(RM) *.o # remove objetos intermediários fclean: clean $(RM) $(NAME) # remove o executável também re: fclean all # recompila do zero
FLAGS DO COMPILADOR GCC
⚠️

-Wall

Ativa todos os avisos comuns: variáveis não usadas, comparações suspeitas, funções sem retorno, etc.

🔍

-Wextra

Avisos extras além do -Wall: parâmetros não usados, comparações entre tipos com/sem sinal, inicializações parciais.

🚫

-Werror

Converte TODOS os avisos em erros. O código não compila enquanto houver qualquer aviso. Padrão Escola 42.
